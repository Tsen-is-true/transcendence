# Frontend Epic 01: 로그인/인증 UX

> 로그인, 회원가입, 인증 상태 유지, 보호 라우트, 42 OAuth 프론트 연동

## 목표

- 이메일/비밀번호 로그인 및 회원가입 UX 완성
- access token 만료 시 프론트 자동 복구
- 42 OAuth 진입과 콜백 처리 완성
- 보호 페이지 공통 가드 정리
- 프론트 기준 검증 규칙과 인증 상태 관리 일원화

## 후속 티켓

1. `FE-AUTH-01` refresh 응답 파싱 및 토큰 재시도 복구
2. `FE-AUTH-02` AuthContext 초기화/로그아웃 흐름 정리
3. `FE-AUTH-03` 보호 라우트 공통 가드 도입
4. `FE-AUTH-04` 42 OAuth 로그인 진입 UI 추가
5. `FE-AUTH-05` 42 OAuth 콜백 페이지 및 토큰 저장 처리
6. `FE-AUTH-06` 로그인/회원가입/설정 검증 규칙 일원화
7. `FE-AUTH-07` 레거시 세션 인증 코드 제거
8. `FE-AUTH-08` 프론트 인증 회귀 테스트 기반 추가

## 권장 작업 순서

## 커밋 원칙

- 한 커밋에는 한 책임만 담는다
- UI 변경과 상태관리 변경은 가능한 한 분리한다
- 라우트 추가와 페이지 구현은 분리한다
- 정리용 삭제 커밋은 기능 커밋과 섞지 않는다
- 테스트는 가능하면 마지막 검증 커밋으로 분리한다
